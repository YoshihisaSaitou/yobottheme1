# タスク

## ドキュメント作成

- `README.md` にテーマ概要、主要ファイルの役割、ディレクトリ構成、運用ルールを記載する
- `spec/requirements.md` を作成し、目的、対象読者、要件、制約、非ゴールを整理する
- `spec/design.md` を作成し、ファイル構成、読み込み設計、命名規則、レイアウト方針を整理する
- `spec/acceptance.md` を作成し、完了条件を定義する

## 内容整理

- ルート直下ファイルの責務を文書化する
- `assets/css`、`assets/js`、`patterns`、`templates`、`parts` の対応関係を文書化する
- `cssMinify()` の対象範囲と除外対象を文書化する
- `theme.json` に依存するレイアウト基準を文書化する
- 未使用ファイルの扱いを明記する

## 整合確認

- ドキュメント記述と現行ファイル構成が一致することを確認する
- `init.php` の読み込み内容と説明文の整合を確認する
- `templates` / `parts` / `patterns` の呼び出し関係が正しいことを確認する
- 用語の表記ゆれを修正する

## 完了確認

- README と `spec/` がベースライン仕様として参照できる状態であることを確認する
- `spec/acceptance.md` の項目を満たすことを確認する
