# 要件定義

## 目的

このテーマの構成、各ファイルの責務、読み込み関係、運用ルールを文書化し、今後新規テーマを作成する開発者と保守担当者が同じ前提で作業できるようにする。

## 対象読者

- 今後この初期テンプレートをベースにテーマを作成する開発者
- 現行テーマを保守・改修する担当者

## 要件

### 構成理解

- テーマ直下の主要ファイルの用途が分かること
- `assets/`、`patterns/`、`templates/`、`parts/` の責務が分かること
- `patterns`、`templates`、`parts`、CSS、JavaScript の対応関係が把握できること

### 責務分離

- `style.css` はテーマメタデータ専用で、スタイルを記述しない方針であること
- 共通処理は `init.php` に集約すること
- テーマ独自の機能やプラグイン連携の入口は `functions.php` であること
- `style.min.css` は自動生成物として扱うこと
- 管理画面専用 CSS とフロント用 CSS を分けて管理すること

### 命名規則

- ブロックパターン関連ファイルは `ブロックパターン名-custom-block-pattern-ナンバー` で命名すること
- パターン本体、CSS、JavaScript は対応関係が分かる命名にすること

### アクセシビリティとレイアウト基準

- 基本単位として `rem` を使用し、ブラウザの基準フォント変更に追従できること
- 通常幅は `960px`、幅広は `1440px` を基本値とすること
- 本文系フォントサイズは原則 `1rem` とすること
- スマホ時の文字サイズは `theme.json` の fluid typography に従うこと
- ブロック余白は基本的に既定の運用値を維持すること

## 対象ファイル・領域

- `README.md`
- `spec/requirements.md`
- `spec/design.md`
- `spec/tasks.md`
- `spec/acceptance.md`

## 制約

- 文書は日本語で作成する
- 現行実装と矛盾しない内容にする
- 実装済みファイルの構成と読み込み関係を前提に記載する

## 非ゴール

- テーマ挙動の変更
- 新規機能の追加
- CSS やテンプレートの実装修正
- `functions.php`、`init.php`、`theme.json` などのコード変更
